## 1. 概述

前端 MVP 收尾迭代要求后端补充 3 个新接口 + 1 个协议变更 + 1 个数据库字段。本设计按 go-zero 三层架构（handler → logic → model/rpc）落地，严格遵循前后端契约文档。

**不做事项**：

- 接口 4 `GET /api/v1/me`（下次迭代）
- Interaction 服务本次**只写 proto**，不实现，`isLiked/isFavorited` 与收藏列表查询返回 stub 值
- Media 服务本次**不实现 RPC logic**，但 gateway 会接入现有 streaming proto，待 Media 实现后即可跑通

## 2. 接口契约

### 2.1 接口 1：GET `/api/v1/users/:userId/posts`

- **权限**：公开（无 JWT）
- **查询参数**：`page`（≥1）、`pageSize`（≤50）、`sortBy`（1=最新, 2=热门，默认 1）
- **响应**：`GetPostListResp`（复用），每条 `PostItem` 填充 `isLiked / isFavorited`（本次硬编码 false）
- **业务规则**：
  1. 只返回 `status=1` 的帖子
  2. 按 `sortBy` 排序（最新=`created_at DESC`；热门=`like_count DESC, created_at DESC`）
  3. 软删除不返回
  4. 目标 userId 不存在时返回业务错误

### 2.2 接口 2：GET `/api/v1/users/:userId/favorites`

- **权限**：基于目标用户 `favorites_visibility` 字段控制：
  - 请求者 == 目标用户 → 始终允许
  - 目标用户 `public (=1)` → 允许
  - 目标用户 `private (=2)` 且请求者 ≠ 目标 → 403
- **响应**：`GetPostListResp`
- **业务规则**：
  1. 按收藏时间倒序
  2. 被软删除的原帖服务端过滤
  3. 每条 `PostItem` 填充请求者的 `isLiked / isFavorited`（本次硬编码 false）
- **本次实现说明**：Interaction 未实现，`GetFavoriteList` 返回空列表，接口返回 `{list: [], total: 0}`

### 2.3 接口 3：POST `/api/v1/media/image`（协议变更）

- **权限**：必须登录（JWT）
- **Content-Type**：`multipart/form-data`（废弃旧 JSON 协议）
- **字段**：`file`（二进制文件）
- **限制**：
  - 单文件 ≤ 10MB
  - Content-Type 白名单：`image/jpeg`、`image/png`、`image/webp`
- **响应**：`{code: 0, data: {url, mediaId, thumbnailUrl}}`
- **实现路径**：gateway handler 手写 multipart 解析 → 通过 streaming 调 Media RPC

## 6. 各层实现设计

### 6.1 Content 层

#### 6.1.1 `GetPostsByIds`（新增）

- 文件：`app/content/internal/logic/get_posts_by_ids_logic.go`

#### 6.3.2 `GetUser` logic 改造

- 移除 `ctxUserId != req.UserId` 的 403 限制
- 从 RPC 响应取 `FavoritesVisibility`，按映射规则填 `FavoritesVisible bool`

#### 6.3.3 `GetUserPosts` logic

- 可选拿 `ctxUserId`（未登录为 0）
- 调 `ContentService.GetUserPosts(userId, page, pageSize, sortBy)`
- 组装 `PostItem`，`isLiked/isFavorited` 本次硬编码 false
- 如果 Content 返回"用户不存在"错误，透传

#### 6.3.4 `GetUserFavorites` logic

- 可选拿 `ctxUserId`（未登录为 0）
- 调 `UserService.GetUser(targetUserId)` 拿 `FavoritesVisibility`
- 权限判断：
  - `ctxUserId == targetUserId` → 允许
  - `FavoritesVisibility == 1`（公开） → 允许
  - 否则 → 返回 403 错误
- **本次 Interaction 未实现**：返回 `{list: [], total: 0, page, pageSize}`
- 后续接入顺序（代码注释标记 TODO）：
  1. 调 `InteractionService.GetFavoriteList(targetUserId, page, pageSize)` 拿 post_ids
  2. 调 `ContentService.GetPostsByIds(post_ids)` 拿帖子
  3. （可选）调 `BatchCheckLiked/BatchCheckFavorited` 填充状态

#### 6.3.5 `UploadImage` handler + logic

**Handler 改造**（`app/gateway/internal/handler/image/upload_image_handler.go`）：

```go
func UploadImageHandler(svcCtx *svc.ServiceContext) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        if err := r.ParseMultipartForm(10 << 20); err != nil {
            httpx.ErrorCtx(r.Context(), w, err)
            return
        }
        file, header, err := r.FormFile("file")
        if err != nil {
            httpx.ErrorCtx(r.Context(), w, err)
            return
        }
        defer file.Close()

        // Content-Type 白名单校验
        ct := header.Header.Get("Content-Type")
        if !isAllowedImageType(ct) {
            httpx.ErrorCtx(r.Context(), w, errx.NewWithCode(errx.UnsupportedImageType))
            return
        }

        l := image.NewUploadImageLogic(r.Context(), svcCtx)
        resp, err := l.UploadImage(file, header)
        if err != nil {
            httpx.ErrorCtx(r.Context(), w, err)
            return
        }
        httpx.OkJsonCtx(r.Context(), w, resp)
    }
}
```

**Logic 改造**：

- 从 ctx 拿 userId
- 调 `MediaService.UploadImage` streaming：
  1. 首包发 `UploadMeta{user_id, file_name, quality, max_width, max_height}`
  2. 循环读取 1MB 块，发 `chunk`
  3. `CloseAndRecv` 拿 `MediaInfo`
- 返回 `UploadImageResp{MediaId, Url, ThumbnailUrl}`

## 7. 错误处理

| 场景 | 错误码 | 说明 |
|---|---|---|
| 目标用户不存在 | `errx.UserNotFound`（新增或复用） | 接口 1、2 |
